XDG

Base Directory Standard

The XDG Base Directory Standard provides a clean and robust specification for storing and retrieving user specific and system wide application configuration and other support files.

What is this all about?

The XDG Base Directory Standard provides a common means for specifying one or more base directories relative to which support files can be located. Using the XDG standard, applications can find and store application specific support files (eg. config files, data caches, etc.) in a universal, clean and robust fashion. The advantages of this standard is that all such files will be found in central locations easily descernable by man and machine. And, if widely adopted, XDG can clear up our HOME directories of all those pesky hidden files.

See XDG Base Directory Standard Specification.

Who's using the XDG Standard?


Why should I use the XDG standard?

If your program depends on user-relative or sytem-wide support files, it will be much improved by using the XDG Base Directory Standard. Not only does it provide a clean yet flexible standard, it also makes your life easier. The #find and #select methods make it a snap to locate support files.

XDG make it easy to enable your applicaiton's users to create user-centic or system-centric configuration and support files, and store those files where they see fit --whether in the standard locations or locations of personal preference.

How to Install the XDG Ruby Library

Using RubyGems

      $ sudo gem install xdg

How to Use the XDG Ruby Library

It's really quite easy, since these are all the methods:

      require 'xdg'

      XDG.config.home
      XDG.config.dirs
      XDG.config.find(pattern){ |path| ... }
      XDG.config.select(pattern){ |path| ... }

      XDG.data.home
      XDG.data.dirs
      XDG.data.find(pattern){ |path| ... }
      XDG.data.select(pattern){ |path| ... }

      XDG.cache.home
      XDG.cache.find(pattern){ |path| ... }
      XDG.cache.select(pattern){ |path| ... }

You can also include XDG into a class and you will get all these methods as well.

      class X
        include XDG
      end
      x = X.new
      x.config.home

See the API documentation for more details.
