XDG

RDoc Rubyforge TigerOps

The XDG specification defines where supporting files should be looked for by defining one or more base directories relative to which files should be located. The advantages of this standard is that files are found in central locations easily descernable by man and machine and (if widely adopted) will clear up our HOME directories of those 10,000 pesky hidden files!

See XDG Base Directory Standard Specification.

Usage

XDG is...

      XDG.config_home
      XDG.config_dirs
      XDG.data_home
      XDG.data_dirs
      XDG.cache_home

      XDG.data_file(file)
      XDG.config_file(file)
      XDG.cache_file(file)

      XDG.config_work
      XDG.data_work
      XDG.cache_work

If you include XDG into a class you will get all these methods but prefixed with xdg_. 

      class X; include XDG; end
      x = X.new
      x.xdg_config_home

If you're not using the XDG standard... well... you suck!

No, seriously, unless you have a seriously better solution, you should be using this standard.

Installation

RubyGems

      $ gem install xdg

Tarball

Installing the tarball requires Ruby Setup (see http://setup.rubyforge.org).

      $ tar -xvzf xdg-0.1.0
      $ cd xdg-0.1.0
      $ setup.rb all

Development

XDG has a git repository on Rubyforge:

You can use gitweb to browse the 'xdg' repository.

To pull the 'xdg' repository anonymously, use:

      git clone git://rubyforge.org/xdg.git

Developers: You can push to the 'xdg' repository using:

      gitosis@rubyforge.org:xdg.git
